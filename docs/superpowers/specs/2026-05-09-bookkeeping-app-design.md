# 设计规范与架构：专业记账 App (Flutter)

## 1. 项目愿景与目标
构建一款面向中文用户的专业记账应用程序。该应用旨在平衡“日常极速记账”与“详尽财务管理”的需求，采用极其现代、美观且富有氛围感的增强版 Material You 设计语言，为用户提供既赏心悦目又高度实用的记账体验。

## 2. 核心架构与数据模型
为了满足“专业详尽”的定位，系统需要支持以下复杂的数据结构：

- **多账本 (Ledgers)**：支持创建多个独立的账本（如：日常开销、旅行专项、装修专款）。
- **多账户 (Accounts)**：支持管理不同的资金来源（如：微信钱包、支付宝、招商银行信用卡、现金）。
- **分类体系 (Categories)**：支持支出/收入的分类，采用“大类 - 子类”的两级结构（如：餐饮-外餐，餐饮-买菜）。
- **账单记录 (Transactions)**：
  - 核心必填字段：金额、类型（支出/收入/转账）、日期、账户、分类。
  - 扩展选填字段：备注 (Notes)、标签 (Tags)、关联图片（可选，未来扩展）。
- **状态管理**：推荐引入 `Riverpod` 作为应用的状态管理方案。由于财务数据需要在明细流、统计图表、预算卡片等多个页面间保持实时同步，响应式的高级状态管理框架是必不可少的。

## 3. UI/UX 设计语言：增强版 Material You
基于提供的 `@designer.md` 指南，我们将严格执行并增强 Material Design 3 的规范：

- **色彩与背景层级 (Tonal Surfaces)**：
  - 绝对禁用纯白 `#FFFFFF` 作为背景。
  - 基础应用背景使用带有暖色调的偏白 `#FFFBFE`。
  - 卡片容器使用 `Surface Container`（如 `#F3EDF7`）以形成自然的层级景深。
- **氛围感与光效 (Atmospheric Effects)**：
  - 在页面头部、关键卡片后方使用大尺寸、高斯模糊 (`blur-3xl`) 的半透明彩色图形（如紫色/粉色圆环）进行叠加，打破死板的网格感。
- **形状与圆角 (Organic Shapes)**：
  - 按钮强制使用胶囊形 (`rounded-full`)。
  - 标准卡片使用 `24px` 圆角。
  - 底部弹窗和大型容器使用 `32px` 或更大的圆角。
- **微交互与动效 (Micro-interactions)**：
  - 所有可交互元素按下时必须带有缩小反馈 (`active:scale-95`)。
  - 重要信息的展示（如：本月结余更新）带有平滑的数字滚动或微光渐变效果。
  - 动画时长保持在 `300ms`，使用标准 MD3 缓动曲线 `cubic-bezier(0.2, 0, 0, 1)`。

## 4. 核心页面规划

### 4.1 首页：经典明细流
- **顶部摘要**：展示当月结余或预算剩余进度，卡片设计具有发光和阴影多层堆叠的质感。
- **时间线列表**：
  - 以“天”为单位对账单进行分组。
  - 每一天是一个大卡片容器，内部按时间倒序排列当天的所有流水。
- **悬浮按钮 (FAB)**：位于右下角，使用应用的主题强调色，点击后唤出记账面板。

### 4.2 记账交互：渐进式底部面板 (BottomSheet)
为了在“不离开当前页面（快捷）”与“输入详尽信息（专业）”中取得完美平衡，采用**渐进式 BottomSheet**：

- **初始状态（快速记账）**：半屏弹起。展示数字键盘、快捷分类横向滚动条、默认账户以及金额输入框。用户可在此状态下 3 秒内完成一笔常规记录。
- **扩展状态（详尽记录）**：当用户在面板内上滑，或点击面板顶部的“更多选项”拉手时，面板扩展至全屏的 80%-90%，暴露出隐藏的详细设置项（切换账本、添加标签、输入详细备注、选择精确到二级子类的分类树）。

## 5. 接下来实施重点
1. **基础脚手架**：清理默认的 Counter App。
2. **主题引擎开发**：配置全局的 Material 3 ThemeData，定义 SeedColor，封装发光模糊背景 (AtmosphericBackground) 和统一卡片 (TonalCard) 组件。
3. **状态层搭建**：设计并在 Riverpod 中初始化 Mock 账单数据模型，跑通数据流。
4. **UI 还原**：实现首页结构，随后攻克高复杂度的渐进式记账 BottomSheet。
